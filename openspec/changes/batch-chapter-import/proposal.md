# 需求说明书

## 背景概述

当前 AI-NovelFlow 平台的章节创建仅支持手动填写表单（标题、序号、内容）逐个创建。对于拥有数十甚至上百章的网络小说而言，逐章录入效率极低。许多用户在导入小说时已有完整的 TXT 文本文件，希望能够一键解析并批量导入所有章节。

## 变更目标

- 支持通过上传 TXT 文件批量导入小说章节，大幅减少手动录入成本
- 提供导入预览功能，让用户在确认前看到解析结果（章节列表、字数、新增/替换状态）
- 兼容中文"第X章 标题"和英文"Chapter X: 标题"两种常见章节格式
- 兼容 UTF-8 和 GBK 编码的 TXT 文件
- 导入时按章节号匹配，已存在的章节替换内容，不存在的章节新建
- 替换已有章节时保留其已生成的角色、分镜、视频等资源（不自动清除）
- 支持部分成功容错，解析或创建失败的章节单独记录，不影响其他章节

## 功能范围

### 新增功能

| 功能标识 | 功能描述 |
| --- | --- |
| `batch-chapter-import` | TXT 文件上传、编码检测、章节解析、预览确认、批量导入完整流程 |

### 修改功能

| 功能标识 | 变更说明 |
| --- | --- |
| `chapter-crud` | 新增批量导入接口，替换已有章节时不清除已生成资源（当前删除章节时会清除资源，导入替换场景需保留） |

## 影响范围

- **代码模块**：
  - 后端: `backend/app/api/chapters.py`（新增两个端点）、`backend/app/repositories/chapter_repository.py`（新增 bulk_upsert 方法）、`backend/app/utils/text_utils.py`（新增文本解析工具）
  - 前端: `frontend/my-app/src/pages/NovelDetail/`（新增批量导入按钮、预览弹窗、状态管理）、`frontend/my-app/src/api/chapters.ts`（新增 API 方法）
- **API接口**：
  - 新增 `POST /api/novels/{novel_id}/chapters/batch-import/preview`（预览解析）
  - 新增 `POST /api/novels/{novel_id}/chapters/batch-import`（执行导入）
- **依赖组件**：无需新增外部依赖（编码检测使用 UTF-8 → GBK 回退策略，无需 chardet）
- **关联系统**：不影响 ComfyUI、LLM 等外部服务

## 验收标准

- [ ] 用户可在 NovelDetail 页面通过"批量导入"按钮上传 TXT 文件
- [ ] 上传后显示预览列表，包含章节序号、标题、字数、操作类型（新增/替换）
- [ ] 正确解析 UTF-8 和 GBK 编码的 TXT 文件，无乱码
- [ ] 正确识别"第X章 标题"格式（含中文数字和阿拉伯数字）
- [ ] 正确识别"Chapter X: 标题"格式
- [ ] 中文数字能准确转换为阿拉伯数字（如"第一百二十三" → 123）
- [ ] 确认导入后，已有章节按 number 匹配替换内容，不存在的章节新建
- [ ] 替换已有章节时，保留其已生成的资源（parsed_data、shot_images、shot_videos 等）
- [ ] 部分章节解析或创建失败时，成功的章节正常入库，失败项单独记录并返回
- [ ] 导入完成后刷新章节列表，显示导入结果统计（成功数、失败数、错误详情）
- [ ] 新增/修改的翻译 key 在 5 种语言文件中均有对应条目
